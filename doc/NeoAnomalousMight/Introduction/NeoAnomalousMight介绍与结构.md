# NeoAnomalousMight Introduction

(一) NeoAnomalousMight是一个基于MinecraftForge1.20.1的Mod

- 1.角色系统（RoleSystem）：我们为这个模组添加了个角色系统，在玩家进游戏存档后，可以选择角色进行游戏（必须选择）

- 2.元素系统（Element）：他是角色系统（RoleSystem）的前置，角色的属性主要来自于他

- 3.新增的BUFF 和 DEBUFF

（重要）基础属性

- 等级：有等级相克的决定，比如30级的玩家或怪物，去面对比他等级高的怪物则伤害会有伤害衰减等，相反....
- 例如等级：玩家（友方）等级50级；敌方等级40级，则友方伤害会增加5%左右。
- 相反，则玩家（友方）等级40级；敌方等级50级，则友方伤害会增加-5%左右

(三)元素（Element）详细设计文档

Q1.1：我目前设想的所有元素类型

- 火（Pyro）高爆发与燃烧
- 水（Hydro）持续性伤害与控制
- 风（Anemo）扩散与聚怪
- 雷（Electro）高频打击与感电
- 草（Dendro）新生与绽放
- 冰（Cryo）控制与爆发
- 岩（Geo）防御与护盾

Q1.2：元素相克关系

- 火 克制 草、冰、水：火属性方伤害+15%（根据上方等级决定）；假如水元素角色玩家的等级在50，去打火元素角色玩家（或者怪物时）则相反
- 水 克制 岩：水属性方伤害+10%，岩属性方防御-10%
- 雷 克制 水：雷属性方伤害+10%，水属性方防御-10%
- 风 增强 火：火属性方伤害+15%，防御-10%，风属性方不变
- 水 增强 雷：雷属性方伤害+10%，防御-5%，闪避-5%
- 岩 克制 雷：岩属性方防御+10%；雷属性方伤害-10%
- 冰 克制 草：草属性方伤害-5%，防御-5%，闪避-5%，冰属性方伤害+10%
- 等等

Q1.3：元素影响角色的基础属性（随等级提升）

- 火系角色伤害+10%
- 水系角色法力+10%
- 风系角色闪避几率和移速+5%
- 雷系角色暴击几率+10%，暴击伤害+50
- 草基础生命值+10%
- 冰伤害+5%，闪避-5%，防御-5%，移速+10%
- 岩防御+15%，移速-2%，闪避几率-4%

Q1.4：没有存在什么元素结合（如：火和水结合的角色）

Q1.5角色或武器拥有和施加元素，都是通过NBT（Named Binary Tag）来实现，它具有保存自定义属性、实体状态、数据；也有储存（.dat,.mca）

（四）角色（role）系统设计概念和理解

Q2.1：我安排的角色目前只有

- 苏林（Su Lin）
- 苏皖（Su Wan）
没有角色类型等说法，也就是说类型都是由元素（Element）定义的

Q2.2：角色职业的大致特点

- 苏林：擅长物理攻击和法术（也就是说，在攻击的时候会有物理和法术伤害，后期计划是可选择法术/物理培养（指的是装备、服饰等））；元素属性暂定
- 苏皖：擅长法术（即法术伤害）；元素属性暂定
由于我们已经设定了基础属性（attributes[ModAttributes.java]）所以“低防高攻”这些的无法实现，我看后面的发展时候需要更改（特别需要）

Q2.3：角色属性成长方向设定

是由等级与品质（Level and Quality）决定的，目前暂时使用固定加值，且等级与品质（Level and Quality）是独立模块（也就是独立一个软件包），在后期（指的是角色/武器/盔甲/服饰/特殊物品）需要引用，且很重要。所以我们团队暂时设定为独立模块

Q2.4：角色的元素属性设定与介绍

元素（Element）使用能力系统（Capabilities）来附加独立的数据对象，有可能我们之前的模块（或者说软件包）设定错误或者没理解
亲和度也就是我们生活中常见的，火对火，伤害很小，火对水，则伤害很高（或者其他），我们推荐使用原版的Custom Attributes，但是我本人对此有点不理解，所以我们看重性能优化和应用快这一方来选择

Q2.5：关于技能树/天赋系统的设定方向和理念

目前我们还没有做出决定是否要添加此系统（或独立模块、子模块）
理念是为角色的基础属性和其他增强+++，如果是这样，那我们游戏将会十分不平衡（后期决定调整怪物数值和其他数值）；关于天赋，这类我们没有想好所以摄入“暂定模块添加”
